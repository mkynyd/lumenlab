# DeepSeek 错误码

| HTTP 状态码 | 错误名称 | 原因 | 解决方案 |
|------------|----------|------|----------|
| **400** | Format Error | 请求体格式错误 | 根据错误信息修正请求体 |
| **401** | Authentication Failure | API Key 不正确 | 检查 API Key；如未创建则创建 |
| **402** | Insufficient Balance | 账户余额不足 | 检查余额并充值 |
| **422** | Parameter Error | 请求体参数无效 | 根据错误信息调整参数 |
| **429** | Rate Limit Reached | 请求速率（TPM 或 RPM）达到上限 | 合理规划请求频率 |
| **500** | Server Fault | 服务器内部错误 | 等待后重试；持续出现则联系支持 |
| **503** | Server Busy | 服务器负载过高 | 稍后重试 |

## 错误处理建议

应用中应有错误处理逻辑，根据 HTTP 状态码向用户展示不同的友好提示：

- **401**: "API Key 无效，请在设置中更新"
- **402**: "账户余额不足，请充值"
- **429**: "请求频率过高，请稍后重试"
- **500/503**: "DeepSeek 服务暂时不可用，请稍后重试"

## 连接超时

- 请求等待推理开始时，服务端每 10 秒发送 keep-alive 信号
- 如果 10 分钟后推理仍未开始，连接将关闭

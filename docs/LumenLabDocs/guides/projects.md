# 项目管理

项目是 LumenLab 中长期学习任务的容器，把资料、对话和成果围绕一个主题组织在一起。

## 项目是什么

每个项目独立维护：

- 上传的资料与解析结果。
- 多轮对话历史，可随时切换。
- 默认模型与系统提示词。
- 当前被选中作为上下文的资料。

普通聊天没有固定上下文，只依赖当前输入；项目聊天可以引用项目内的资料，回答更贴合你的学习主题。

## 项目类型

| 类型 | 用途 | 默认系统提示词侧重 |
|------|------|------------------|
| 实验 | 实验报告、调试记录 | 步骤化输出、引用资料原文 |
| 复习 | 复习提纲、错题总结 | 结构化整理、易于记忆 |
| 编程 | 代码 review、单元测试 | 代码优先、解释次之 |
| 通用 | 其他学习场景 | 中性回答 |

类型只影响创建时生成的默认提示词和快捷任务，之后你可以在项目内自由调整对话方式。

## 创建项目

入口：侧边栏「项目」→「新建项目」。

需要填写：

- 名称（必填）。
- 类型（实验 / 复习 / 编程 / 通用）。
- 使用场景描述（可选，用于生成更贴合的系统提示词和快捷任务）。

创建后项目默认使用 deepseek-v4-pro 模型并开启深度推理模式。

## 项目资料

支持的资料类型：

- 图片（通过 MiniMax M3 OCR 识别文字）。
- PDF（通过 MiniMax M3 原生文档解析）。
- 文本 / 代码文件（直接读取）。

每份资料会经历以下状态：

1. 解析中。
2. 解析完成，可被对话引用。
3. 解析失败，可重试。

知识增强不会自动执行，如需使用，请在文件详情页点击「知识增强」按钮手动触发。

## 项目对话

- 项目内的所有对话都归属于该项目。
- 对话会根据问题类型决定如何引用资料，例如直接加载小文件全文、由 AI 挑选相关文件，或使用关键词 / 混合检索。
- 单个项目可以创建多个对话，顶栏可快速切换。

> 如需修改项目的默认模型或系统提示词，项目高级配置功能即将上线。

## 快捷任务

快捷任务是一组与项目类型绑定的常用提示模板：

- 以按钮条形式显示在项目聊天输入框上方。
- 点击后自动发送对应的预设提示，快速开始常见任务。

## 项目删除

删除项目需要二次确认，会同时移除项目内的资料、对话和 Artifact。删除后不可恢复，请谨慎操作。

> 「导出项目」功能尚未实现。

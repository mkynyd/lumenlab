## 资料生命周期

上传资料后会进入解析流程：

1. 写入存储并创建资料记录，状态为「解析中」。
2. 解析成功后状态变为「解析完成」，可被对话引用。
3. 解析失败则状态为「解析失败」，可重试或换一份文件上传。

知识增强有独立的状态：未增强 / 增强中 / 已增强 / 过期 / 失败。

## 支持的资料类型

| 类型 | 说明 |
|------|------|
| PNG / JPG / WebP | 自动调用 MiniMax M3 OCR 识别图片中的文字 |
| PDF | 自动调用 MiniMax M3 原生文档解析，不区分文本型还是扫描型 |
| DOC / DOCX / PPT / PPTX / XLS / XLSX | 自动调用 MinerU 转换为 Markdown，并保存文档内图片资源 |
| WPS / ET / DPS | 按 Office 文档处理，适合 WPS 课件和表格 |
| Pages / Numbers / Keynote | 按 iWork 文档处理，适合 macOS 课程资料 |
| Markdown / TXT / CSV / 代码文件 | 直接提取文本内容 |

上传限制：单文件 50MB，单次最多 50 个文件，总大小 300MB。

## OCR 与 PDF 解析

- 图片资料会直接调用 MiniMax M3 视觉理解，得到结构化文本。
- PDF 资料统一调用 MiniMax M3 原生文档解析。
- 解析结果可以在文件详情页查看和简单编辑。

## Office 文档解析

- Office、WPS 与 iWork 文档使用 MinerU 解析为 Markdown。
- 解析过程中提取到的图片会保存为项目文件资源，并在 Markdown 中改写为私有资源链接。
- 该流程需要当前用户或中央凭证组配置 MinerU Token。
- 如果文档主要是扫描图片或视觉排版，后续对话可能会路由到 MiniMax M3，以保留多模态上下文连续性。

## 知识增强

- 在文件详情页点击「知识增强」按钮手动触发。
- 系统会调用 AI 为文件生成增强内容，便于后续检索。
- 全文检索时，如果文件已完成增强，会优先使用增强后的内容。

## 检索策略

系统会根据当前问题自动选择最合适的检索方式：

| 策略 | 触发场景 |
|------|----------|
| 不使用资料 | 普通闲聊，不需要引用项目资料 |
| 全文加载 | 用户明确针对某份小文件提问，且文件内容不超过约 8000 字符 |
| 关键词检索 | 未开启向量检索，或策略选择关键词检索 |
| 混合检索 | 关键词检索与向量检索结果融合 |

检索流程大致如下：

1. 如果你勾选了具体文件，小文件会直接全文加载，大文件则作为候选范围。
2. 如果没有明确勾选，系统会先用项目索引和关键词判断哪些资料可能与问题相关。
3. 在候选资料范围内做关键词或混合检索。
4. 针对全项目语料的任务（如提取知识点、生成复习材料）会直接使用所有已解析或部分解析的资料。

## 向量检索

- LumenLab 使用阿里云百炼 qwen3-vl-embedding 1024 维融合模式生成向量。
- 当前用户或中央凭证组配置百炼 API Key 后，解析文件时会自动生成向量。
- 混合检索会结合向量相似度和关键词匹配，返回更相关的内容。
- 设置页目前没有「向量命中率」指标。

## 资料图谱

项目侧边栏提供「资料图谱」入口。图谱按三类节点展示项目资料：

| 节点 | 含义 |
|------|------|
| topic | 从项目索引和文件摘要中提取的主题 |
| file | 项目中的资料文件 |
| chunk | 可检索的文本分块 |

点击节点可以查看类型、摘要、解析状态和相关文件。资料图谱用于理解资料覆盖范围和解析质量，不会改变原始资料内容。

## 常见问题

- 解析失败：检查文件格式与大小（单文件上限 50MB），查看文件详情页的错误提示。Office/WPS/iWork 文档还需要 MinerU Token。
- 检索为空：确认文件状态为「解析完成」，或在对话中显式勾选文件作为上下文。
- 向量检索未生效：确认当前用户或中央凭证组已配置百炼 API Key。

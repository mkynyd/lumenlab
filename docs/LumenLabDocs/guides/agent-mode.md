# Agent 模式

Agent 模式让 AI 以多步方式完成复杂任务：它可以读取资料、搜索信息、保存 Artifact 等，每一步有副作用的操作都会先征求你的同意。

## 什么是 Agent 模式

Agent 模式是一种受控的多步对话形式。模型可以在该模式下：

- 规划任务步骤。
- 调用工具获取信息或执行操作。
- 把工具执行结果反馈给模型，继续下一步，直到任务完成或被取消。

所有可能产生副作用的操作都会经过策略引擎检查，并在必要时弹出审批请求。

## 与普通聊天的区别

| 维度 | 普通聊天 | Agent 模式 |
|------|----------|------------|
| 工作方式 | 单轮回答 | 可多轮调用工具，直到任务完成 |
| 可执行操作 | 仅模型内置能力 | 可读取项目资料、搜索、保存成果等 |
| 用户介入 | 只在输入时 | 遇到需要审批的操作时会暂停等你确认 |
| 副作用控制 | 无 | 所有副作用都需要经过审批 |

## 审批卡片

当 Agent 模式需要执行一个可能改变状态的操作时，会显示审批卡片，告诉你：

- 要执行什么操作。
- 会影响哪些资料或成果。
- 是否可撤销。

你可以选择：

- 仅本次允许：生成一次性授权，执行当前这次操作。
- 本会话同类允许：当前会话内，同类操作后续可自动执行。
- 拒绝：跳过当前操作，模型会继续尝试完成后续步骤。

## 风险等级

操作按风险高低分为几档：

- 低风险只读操作：例如列出资料、读取文件内容，通常自动执行。
- 会话级副作用：例如保存 Artifact，首次需要审批，当前会话后续可自动。
- 单次高风险操作：例如删除资料、导出文件，每次都会询问你。
- 系统级高风险操作：如有此类操作，会强制每次询问，目前主要用于预留的安全策略。

## 常见问题

- 为什么我看不到独立的 Agent 入口：目前 Agent 事件内联显示在对话中，没有单独入口。
- 审批一直等待怎么办：刷新页面会清空未完成审批，需重新发起对话。
- 点击拒绝后会怎样：当前操作会被标记为失败，模型可继续后续步骤，不会终止整个任务。
- 工具被阻止是为什么：可能是该操作不在当前能力范围内、风险等级超出限制，或参数校验未通过。
- 参数被替换为什么会被拒绝：为保障安全，审批后如果模型变更了操作参数，系统会因参数不一致而拒绝执行。

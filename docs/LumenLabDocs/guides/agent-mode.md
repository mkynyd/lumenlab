# Agent 模式

Agent 模式让 AI 以多步方式完成复杂任务：它可以读取资料、搜索信息、保存 Artifact 等。每次工具调用都会经过策略检查；需要审批的操作会暂停并等待你决定。

## 什么是 Agent 模式

Agent 模式是一种受控的多步对话形式。模型可以在该模式下：

- 通过 Skill Router 选择合适的学习场景指令。
- 规划任务步骤。
- 调用工具获取信息或执行操作，例如读取项目资料、检索网页、保存成果、管理参考文献。
- 把工具执行结果反馈给模型，继续下一步，直到任务完成或被取消。

所有可能产生副作用的操作都会经过策略引擎检查，并在必要时弹出审批请求。

## 与普通聊天的区别

| 维度 | 普通聊天 | Agent 模式 |
|------|----------|------------|
| 工作方式 | 单轮回答 | Skill Router + Agent Runtime + 工具调用 |
| 可执行操作 | 仅模型内置能力 | 可读取项目资料、检索网页/arXiv、保存成果、管理引用等 |
| 用户介入 | 只在输入时 | 遇到需要审批的操作时会暂停等你确认 |
| 副作用控制 | 无 | Policy Engine 按风险、Skill 和用户偏好决定自动、询问或阻断 |

## 审批卡片

当 Agent 模式需要执行一个可能改变状态的操作时，会显示审批卡片，告诉你：

- 要执行什么操作。
- 会影响哪些资料或成果。
- 是否可撤销。

你可以选择：

- 仅本次允许：兑换一次性授权并立即执行当前操作。
- 本会话同类允许：对 L1 / L2 操作立即执行，并让当前会话后续同类操作可自动执行。
- 拒绝：把当前操作标记为已拒绝，不执行 Tool。

批准后，卡片会显示 Tool 的真实 `succeeded` 或 `failed` 结果；拒绝后显示拒绝终态。当前实现不会自动恢复触发审批的那一次模型 continuation，若还需要模型基于结果继续工作，请发送下一条消息。

## 风险等级

操作按风险高低分为几档：

- L1 低风险只读操作：例如列出资料、读取文件、检索项目资料、搜索网页/arXiv，通常自动执行。
- L2 会话级副作用：例如保存 Artifact、新增或挂载参考文献，首次需要审批，当前会话后续可预批准。
- L3 单次高风险操作：例如删除资料、导出 DOCX，每次都会询问你。
- L4 系统级高风险操作：预留的阻断级策略，目前没有生产 Tool 默认开放。

## 来源展示

Agent 工具结果会被规范化为来源并写入消息记录。回答底部会展示项目文件、网页、arXiv 论文或引用条目，便于你核对依据。模型正文不会强行插入编号引用，来源统一由底部 UI 管理。

## 当前边界

- DeepSeek 优先使用原生 Tool；当前不支持原生的内部 Tool 由 ProviderAdapter 使用 XML/DSML fallback，并在展示前清除协议标记。
- MiniMax M3 除了多模态理解，也会以原生 Tool 协议接收当前允许的工具；续跑时不会重复发送首轮附件。
- Qwen3.7-Plus 启用后通过 DashScope 原生多模态与 Function Calling 协议进入同一工具循环，支持文本输出与图像、视频理解。
- DeepSeek / MiniMax 可选用隔离的 `pi-ai` Provider POC；Qwen 不经过该适配层。
- 三条 Provider 路径都进入同一个 AgentLoop 与 ToolRunner，审批、审计和状态持久化规则一致。
- Web 抓取受 allowlist 和 SSRF 校验限制，不保证任意网址都能读取。

## 常见问题

- 为什么我看不到独立的 Agent 入口：目前 Agent 事件内联显示在对话中，没有单独入口。
- 为什么 Skill 没有自动激活：任务可能缺少资料、提问过泛，或你手动关闭了 Skill。可以在输入工具栏手动切换。
- 审批一直等待怎么办：可以在卡片中允许或拒绝；刷新可能移除当前页面中的 Timeline 展示，但待审批记录仍由服务端持久化。
- 点击允许后会怎样：服务端立即执行 Tool，卡片显示成功或失败终态；模型不会自动续跑，可发送下一条消息继续。
- 点击拒绝后会怎样：当前操作会落为 `rejected`，模型同样不会自动续跑。
- 工具被阻止是为什么：可能是该操作不在当前能力范围内、风险等级超出限制，或参数校验未通过。
- 参数被替换为什么会被拒绝：为保障安全，审批后如果模型变更了操作参数，系统会因参数不一致而拒绝执行。

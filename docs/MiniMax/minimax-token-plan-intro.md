# MiniMax Token Plan 概览

> 来源: https://platform.minimaxi.com/docs/token-plan/intro

## 什么是 Token Plan

MiniMax 提供订阅制套餐，覆盖语言、视频、语音、音乐和图片五种模态模型。用户通过专属的**订阅 Key** 使用。

## 订阅 Key

- 每个团队的每个用户拥有一把专属订阅 Key
- 用户在拥有活跃的 Token Plan 席位或积分时，该 Key 生效
- **与按量计费 API Key 不可互换**

## 套餐档位

| 套餐 | 价格 | 典型 Agent 用量 |
|------|------|-----------------|
| Plus | ¥49/月 | 3-4 个 Agent |
| Max | ¥119/月 | 4-5 个 Agent |
| Ultra | ¥469/月 | 6-7 个 Agent |

所有档位使用 **5 小时固定窗口 + 周窗口** 进行额度管理。

## 三大核心卖点

1. 单一用量条覆盖多模态能力
2. 面向长上下文 Agent 和编码工作流设计
3. 可预测的固定定价

## 额度用完后

四种应对方案：
1. 使用已购积分
2. 升级订阅套餐
3. 切换到按量计费 API Key
4. 等待限速窗口重置

未使用的订阅额度不结转。

## 工具集成

支持以下工具的接入指南：
- OpenClaw
- **Claude Code**
- Cursor
- TRAE
- Hermes Agent
- 其他工具

> Tip：推荐使用 MiniMax CLI 访问多模态资源。

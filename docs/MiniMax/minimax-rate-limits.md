# MiniMax 速率限制

> 来源: https://platform.minimaxi.com/docs/guides/rate-limits

## 概述

MiniMax 平台通过 **RPM**（每分钟请求数）和 **TPM**（每分钟 token 数）两个维度进行速率限制，以保障服务可用性和稳定性。

## 相关说明

### 什么是速率限制
API 对用户在指定时间段内访问次数或消耗 token 数的限制。

### 为何有限速
防止滥用和误用，确保公平访问，帮助为所有用户保持流畅一致的体验。

### 限速适用范围
主账号与子账号共同享有所有速率限制配额。例如 RPM 为 120 时，若主账号使用了 30 个请求，子账号该分钟还可发送 90 个请求。

### 触发限速的后果
API 返回速率限制错误并拒绝进一步请求，直到指定时间过后。

### 使用建议
RPM 与 TPM 独立限制。若 RPM 已达上限但 TPM 尚有容量，可将多个任务批量放入每个请求以提高 token 吞吐量。

### 申请提升限速
通过官方客服或邮件 api@minimaxi.com 申请。处理可能需要 3-5 个工作日。

# MiniMax Token Plan 常见问题

> 来源: https://platform.minimaxi.com/docs/token-plan/faq

## 可用套餐

| 套餐 | 价格 | 典型 Agent 用量 |
|------|------|-----------------|
| Plus | ¥49/月 | 3-4 个 Agent |
| Max | ¥119/月 | 4-5 个 Agent |
| Ultra | ¥469/月 | 6-7 个 Agent |

## 支持的资源与模型切换

Token Plan 覆盖开放平台所有模型，不同模型/模态共享同一套套餐额度（非分离额度）。

## 额度共享机制

文本、图片、语音等额度**不是分开的**。所有资源共用一套额度。切换模型时需使用对应 API 或工具接入页面的模型 ID。

## 订阅 Key 说明

- 订阅 Key 是 Token Plan 专属密钥
- 每个团队每位用户一把
- 与普通按量计费 API Key **相互独立，不可混用**
- 订阅前就已存在，被分配席位/积分权限后即可使用

## 已购积分折算

- **折算比例**：1,000 积分 = ¥7（与按量付费目录价等值）
- **扣减方式**：按资源目录价折算为积分扣除
- **覆盖范围**：文本、图像、语音、音乐；视频以控制台为准
- **扣减优先级**：优先扣套餐额度，超出再扣积分

## 积分独立使用

积分可以无订阅席位的情况下单独购买和使用，仍通过订阅 Key 使用。如需覆盖范围之外的资源，使用按量计费 API Key。

## 查看用量

**方式一**：控制台 → 订阅付费 → 套餐用量

**方式二**：API 查询
```bash
curl --location 'https://www.minimaxi.com/v1/token_plan/remains' \
  --header 'Authorization: Bearer <API Key>' \
  --header 'Content-Type: application/json'
```

消耗等级：
- **低**：日常聊天、翻译、简单写作
- **中等**：代码生成、多轮对话
- **较高**：长上下文推理、多模态任务、复杂 Agent 工作流

## 用量重置规则

- 套餐额度：受 5 小时固定窗口 + 周窗口控制
- 未用完的套餐额度不结转
- 已购积分：按自身有效期使用

## 达到上限的应对

1. 使用已购积分（自动补充）
2. 升级订阅套餐（立即生效）
3. 切换按量付费（更换为普通 API Key）
4. 等待窗口重置

## 平台流量规则

高峰时段动态限流（通常工作日 15:00-17:30）：
- Plus：约 3-4 个 Agent
- Max：约 4-5 个 Agent
- Ultra：约 6-7 个 Agent

## TPS 计算

TPS = 输出 token 总数 / (最后一个 token 生成时间 - 第一个 token 生成时间)。各页面标注的 TPS 为参考值，实际使用可能波动。

## 生产环境适用性

Token Plan 面向个人开发者的交互式使用场景，生产环境建议使用按量付费。主要限制：RPM/TPM 限制和 5 小时/周窗口额度。

## 开票规则

- 支付宝直接付款：可开票
- 余额支付：可开票
- 余额 + 支付宝组合：可开票
- 代金券抵扣部分：**不可开票**（仅实际支付金额可开票）
